Access Workflow

How the access service works

1. Account provisioning

Create an account with required identity details, then optionally answer workbench-specific questions so the service can personalize your profile and defaults.

2. Credential entry

Use your email and password to initiate access to the workspace.

3. Human verification

Complete the scribble code and emoji match the first time a user signs in on a device, or again after trusted access expires.

4. Remembered access

If you opt into Remember this device, the workbench stores device trust for that user so later sign-ins can skip verification until the trust window ends.

Profile Service

Personalization surfaces

Public user ID

Each account receives a unique service identifier that can follow run metadata, live tracing context, and future ownership records.

Avatar

You can upload a profile image or fall back to the generated initials badge shown in the top-right workspace header.

Primary system

Your profile can store whether you mainly work on Windows, Linux, or macOS.

Live tracing system preference

This default helps the Live Monitoring service prefill the operating-system context for new live sessions.

Preferred analysis mode

Controls the default result mode used when you launch a new analysis.

Operational Workflow

How the service areas fit together

Overview

Use this page to confirm model readiness, review the latest saved result, and see the current user-scoped system status.

Analysis Service

Run anomaly detection on pasted text, uploaded files, or prepared scenarios. New runs inherit your profile-aware defaults where applicable.

Live Monitoring Service

Monitor an updating file or replay stream while labeling the traced system as Windows, Linux, or macOS so archived snapshots preserve that context.

Run Archive

Saved runs are private to the active account and can carry profile-linked metadata such as public user ID and live-trace context.

Run Details

Each saved run opens with a frozen timeline, evidence table, metadata panel, and export actions for later review.

Metrics Explained

Common terms

Agreement

The share of windows where the baseline sequence model and the argument-aware model produce the same anomaly verdict.

Adaptive threshold

A runtime rule that can adjust the argument-aware model threshold when drift conditions change.

Drift posture

A lightweight signal showing whether recent behavior looks stable or is shifting away from the learned baseline.

Host group

A derived grouping used for proxy cross-host analysis when explicit host identifiers are missing.

Metric wins

The number of tracked benchmark metrics where the argument-aware model outperforms the baseline model.

Cross-host delta

The difference between average proxy host-group accuracy for the argument-aware model and the baseline model.

FAQ

Common questions

Why am I asked to complete a human check?

The access service uses a local human-verification sequence before opening a secured workspace session.

What does Remember this device do?

It creates a trusted-device record for the same user on the same device, which lets later sign-ins skip the human check until that trust expires.

What is the public user ID for?

It provides a stable account identifier that can follow saved run metadata and live service context.

Why does Live Monitoring ask for a system label?

The Windows, Linux, or macOS label preserves operating-system context for live traces and archived snapshots.

Can I change my avatar later?

Yes. The Profile service supports avatar upload, replacement, or a return to initials-based presentation.

Troubleshooting

Quick checks

Access challenge keeps failing

Refresh the challenge if the scribble card is unclear or if the active verification window has expired.

Avatar is not appearing

Use a supported image type such as PNG, JPG, WEBP, or GIF and keep the file within the configured size limit.

No result appears after analysis

Check that models have finished bootstrapping and that the input contains enough records to build windows.

Live monitor shows no change

Make sure the selected file path exists, the correct system label is selected, and the file is actively receiving new lines.

Evaluation is still pending

The benchmark worker may still be running in the background. Refresh the page after a short wait.
