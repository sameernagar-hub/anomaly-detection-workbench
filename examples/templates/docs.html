App Services

What each window does

Overview

High-level operational status and access point to the rest of the app.

Analysis Service

Runs inference on text, files, and prepared sample scenarios.

Live Monitoring Service

Monitors a changing file or replay stream without replacing archived run details.

Evaluation Service

Shows benchmark metrics and proxy cross-host comparison results.

Run Archive

Stores bootstrap runs, analysis runs, and archived live snapshots during the current session.

Run Details

Shows one saved run with a frozen timeline, table, metadata, and export actions.

Metrics Explained

Common terms

Agreement

The share of windows where the baseline model and improved model produced the same verdict.

Adaptive threshold

A runtime rule that can adjust the improved-model threshold when conditions suggest drift.

Drift posture

A lightweight signal showing whether current behavior looks stable or is moving away from the learned baseline.

Host group

A derived grouping used to cluster records by host-like source when explicit host fields are missing.

Metric wins

The number of benchmark metrics where the improved model outperforms the baseline.

Cross-host delta

The difference between average proxy host-group accuracy for the improved model and the baseline model.

FAQ

Common questions

Why does the live graph keep changing?

The live page is designed for updating inputs. Archive a snapshot when you want a stable result page.

Why is the run archive empty after restart?

The current archive is stored in memory for the active Flask session only.

What is the difference between a run and a snapshot?

A run is any saved analysis result. A snapshot is a live-monitor state that has been archived as a run.

When should I use Evaluation Service?

Use it when you want benchmark comparison metrics, not when you need to inspect one operational run.

Troubleshooting

Quick checks

No result appears after analysis

Check that models have finished bootstrapping and that the input contains enough records to build windows.

Live monitor shows no change

Make sure the selected file path exists and is actively receiving new lines.

Evaluation is still pending

The benchmark worker may still be running in the background. Refresh the page after a short wait.
