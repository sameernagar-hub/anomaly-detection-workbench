Metrics Explained

Common terms

Agreement

The share of windows where the baseline sequence model and the argument-aware model produce the same anomaly verdict.

Adaptive threshold

A runtime rule that can adjust the argument-aware model threshold when drift conditions change.

Drift posture

A lightweight signal showing whether recent behavior looks stable or is shifting away from the learned baseline.

Host group

A derived grouping used for proxy cross-host analysis when explicit host identifiers are missing.

Metric wins

The number of tracked benchmark metrics where the argument-aware model outperforms the baseline model.

Cross-host delta

The difference between average proxy host-group accuracy for the argument-aware model and the baseline model.

FAQ

Common questions

Why does the live graph keep changing?

The live page is designed for updating inputs. Archive a snapshot when you want a stable result page.

Why is the run archive empty after restart?

The current archive is stored in memory for the active Flask session only.

What is the difference between a run and a snapshot?

A run is any saved analysis result. A snapshot is a live-monitor state that has been archived as a run.

When should I use Evaluation?

Use it when you want benchmark comparison results, not when you need to inspect one specific anomaly detection run.

Troubleshooting

Quick checks

No result appears after analysis

Check that models have finished bootstrapping and that the input contains enough records to build windows.

Live monitor shows no change

Make sure the selected file path exists and is actively receiving new lines.

Evaluation is still pending

The benchmark worker may still be running in the background. Refresh the page after a short wait.

Why are drift or adaptive values not changing?

Those signals need enough windows to compare recent behavior against earlier behavior in the same run.
